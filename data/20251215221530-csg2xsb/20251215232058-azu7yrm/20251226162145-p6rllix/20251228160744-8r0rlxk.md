# 一、核心动词的选用

# AI给灵感、你筛选

动词不是“动作本身”，而是**包含各种二阶信息——它不只是描述动作，还“顺带”把场景、伦理姿态、声音、时间质地一起编码进去了。**

- 僧推月下门→僧敲月下门
- 躺在沙发上→瘫在沙发上

 **<u>——动词本身，就在替你写人物与剧情走向。</u>** 选中一个动词，就等于选中了该动词的“默认情境包”。

‍

靠自己瞎想，怎么都想不出多好的来......还是让AI尽可能多给些灵感，你挑选

###### AI给灵感的提示词

```undefined
任务：把一个基础动作扩展为“候选动词库”（用于写作选词）。

基础动作：{在此输入，如：推门 / 躺 / 歪头 / 踮脚}
数量：{60 或 80}

输出规则（必须严格遵守）：
- 输出内容只能是“候选动词”；不要任何解释、不要标签、不要分组标题、不要序号、不要标点、不要空行。
- 每行只输出 1 个动词。
- 只输出“单个动词词语”（1–2字为主，允许少量3–4字成词）；禁止出现宾语/介词结构/连词；禁止动词短语。
- 为避免短语化：禁止“动词+补语/趋向”结构（如：推开、走进、躺下、站起、抬起等）。
- 所有候选必须与基础动作属于同一动作类型，但要尽量覆盖不同“内含信息”的动词（力度/速度/持续/阻力/路径/姿态/情绪/礼貌-越界/公开-隐秘/主动-被动等）；覆盖规划在内部完成，不要输出规划过程。
- 去重；避免仅一字之差但信息增量极小的近重复。

开始输出。
```
